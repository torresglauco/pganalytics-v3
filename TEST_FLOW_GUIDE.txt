TIMELINE: ~30-45 minutes total

================================================================================
PHASE 1: Infrastructure Setup (10-15 minutes)
================================================================================

Command:  ./cleanup-and-start-load-test.sh

What it does:
  1. Stops existing containers (both docker-compose.yml and load test)
  2. Removes all pganalytics volumes
  3. Cleans Docker resources
  4. Verifies cleanup complete
  5. Starts docker-compose-load-test.yml
  6. Waits for core services to be healthy:
     ✓ PostgreSQL (metadata)
     ✓ TimescaleDB (metrics)
     ✓ Backend API
     ✓ Frontend UI
  7. Displays access URLs

Expected Output:
  Infrastructure started
  Waiting for core services to be healthy...
  ✓ PostgreSQL OK
  ✓ TimescaleDB OK
  ✓ Backend OK
  ✓ Frontend OK

During this phase:
  - 40 Target PostgreSQL instances start automatically
  - 40 Collectors start automatically and begin auto-registering
  - Collectors attempt registration with backend (~1-2 min)

Log patterns to look for:
  [collector-001] Auto-registering collector for the first time...
  [collector-002] Auto-registering collector for the first time...
  ...
  [backend] Registered collector: col_001 (UUID: xxx)
  [backend] Registered collector: col_002 (UUID: xxx)

Warning signs (if you see these, something is wrong):
  - "connection refused" from collectors
  - "backend unhealthy" in status
  - "error: timeout waiting for service"
  → Check: docker logs backend, docker ps, docker network ls

Success indicators:
  - docker ps shows 84 containers
  - curl http://localhost:8080/api/v1/health returns 200
  - curl http://localhost:4000 returns 200

TROUBLESHOOTING QUICK REFERENCE
================================================================================

Container not starting?
  docker-compose -f docker-compose-load-test.yml logs <service_name>

Collector not registering?
  docker-compose -f docker-compose-load-test.yml logs collector-001 | tail -20

Metrics not appearing?
  docker-compose -f docker-compose-load-test.yml logs backend | grep -i metric

API not responding?
  curl http://localhost:8080/api/v1/health

Too many containers?
  docker-compose -f docker-compose-load-test.yml down -v

Need fresh start?
  ./cleanup-and-start-load-test.sh
