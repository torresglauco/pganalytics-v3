#### Subtask 1.1.1: StatefulSet for Backend (6 hours)
**Owner**: DevOps Engineer
**Description**: Create Kubernetes StatefulSet for pgAnalytics backend API

**Requirements**:
- 3 replicas (configurable)
- Resource limits: CPU 500m, Memory 512Mi
- Resource requests: CPU 250m, Memory 256Mi
- Health checks: liveness, readiness, startup probes
- Init containers for database migrations
- Volume mounts for TLS certificates
- Service account and RBAC

**File**: `kubernetes/backend-statefulset.yaml`

**Acceptance Criteria**:
- [ ] StatefulSet YAML is valid (helm lint passes)
- [ ] 3 replicas specified
- [ ] All health probes configured
- [ ] Init container for migrations included
- [ ] TLS volume mounts configured
- [ ] Resource limits set correctly
- [ ] Service account created

**Tasks**:
1. Create namespace definition
2. Create service account
3. Create StatefulSet definition
4. Configure 3 replicas
5. Setup health probes
6. Configure init containers
7. Add TLS volume mounts
8. Test YAML syntax

**Code Template**:
```yaml
apiVersion: apps/v1
kind: StatefulSet
metadata:
  name: pganalytics-backend
  namespace: pganalytics
spec:
  serviceName: pganalytics-backend
  replicas: 3
  selector:
    matchLabels:
      app: pganalytics-backend
  template:
    metadata:
      labels:
        app: pganalytics-backend
    spec:
      serviceAccountName: pganalytics-backend
      initContainers:
      - name: migrate
        image: pganalytics/api:3.3.0
        command: ["./pganalytics-api", "migrate", "up"]
      containers:
      - name: api
        image: pganalytics/api:3.3.0
        ports:
        - containerPort: 8080
          name: http
        livenessProbe:
          httpGet:
            path: /api/v1/health
            port: 8080
          initialDelaySeconds: 30
          periodSeconds: 10
        readinessProbe:
          httpGet:
            path: /api/v1/health
            port: 8080
          initialDelaySeconds: 10
          periodSeconds: 5
        resources:
          limits:
            cpu: 500m
            memory: 512Mi
          requests:
            cpu: 250m
            memory: 256Mi
        volumeMounts:
        - name: tls
          mountPath: /etc/tls
          readOnly: true
      volumes:
      - name: tls
        secret:
          secretName: pganalytics-tls
```

**Owner**: DevOps Engineer
**Description**: Create Kubernetes DaemonSet for pgAnalytics collectors

**Requirements**:
- One pod per node
- Auto-registration with backend
- Node affinity for database-adjacent nodes
- Resource limits appropriate for collectors
- Volume mounts for configuration

**File**: `kubernetes/collector-daemonset.yaml`

**Acceptance Criteria**:
- [ ] DaemonSet YAML is valid
- [ ] One pod per node (no node selectors limiting)
- [ ] Auto-registration environment variables set
- [ ] Configuration volume mounts configured
- [ ] Resource limits set
- [ ] Health checks configured

**Code Template**:
```yaml
apiVersion: apps/v1
kind: DaemonSet
metadata:
  name: pganalytics-collector
  namespace: pganalytics
spec:
  selector:
    matchLabels:
      app: pganalytics-collector
  template:
    metadata:
      labels:
        app: pganalytics-collector
    spec:
      containers:
      - name: collector
        image: pganalytics/collector:3.3.0
        env:
        - name: COLLECTOR_ID
          valueFrom:
            fieldRef:
              fieldPath: spec.nodeName
        - name: BACKEND_URL
          value: "https://pganalytics-backend:8080"
        volumeMounts:
        - name: config
          mountPath: /etc/pganalytics
      volumes:
      - name: config
        configMap:
          name: pganalytics-collector-config
```
