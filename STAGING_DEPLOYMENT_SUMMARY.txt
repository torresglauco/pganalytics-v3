🎯 PHASE 4 FEATURES BEING DEPLOYED
═══════════════════════════════════════════════════════════════════════════════

1. Custom Alert Conditions
   • Create flexible alert rules with metric/operator combinations
   • Metrics: error_count, slow_query_count, connection_count, cache_hit_ratio
   • Operators: >, <, ==, !=, >=, <=
   • Time windows: 1-1440 minutes
   • Human-readable condition preview

2. Alert Silencing
   • Temporarily suppress alerts (5 minutes to 24 hours)
   • TTL-based auto-expiration (no cleanup job needed)
   • Context/reason tracking for audit trail
   • Quick deactivation option
   • Active silence list view

3. Escalation Policies
   • Multi-step alert routing (2-5 steps per policy)
   • Channel support: Email, Slack, PagerDuty, Webhook
   • Configurable wait times between steps
   • Acknowledgment tracking
   • Real-time state tracking

🐛 TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

Common Issues and Solutions:

Issue: Docker daemon not running
Solution: Start Docker Desktop or service

Issue: Ports already in use
Solution: Modify ports in docker-compose.staging.yml

Issue: Database connection failing
Solution: Check PostgreSQL logs: docker-compose logs postgres

Issue: API not responding
Solution: Check API logs: docker-compose logs api

Issue: Frontend showing errors
Solution: Check frontend logs: docker-compose logs frontend

For detailed troubleshooting, see:
  • docs/STAGING_DEPLOYMENT_PLAN.md (§ Troubleshooting)
  • STAGING_DEPLOY_QUICK_START.md (§ Troubleshooting)

🔄 ROLLBACK PROCEDURES
═══════════════════════════════════════════════════════════════════════════════

Quick Rollback (< 5 minutes):
  1. Stop services: docker-compose -f docker-compose.staging.yml down
  2. Checkout previous: git checkout v3.4.0
  3. Restart: docker-compose -f docker-compose.staging.yml up -d

Graceful Rollback (10-15 minutes):
  1. Stop API: docker-compose -f docker-compose.staging.yml stop api
  2. Drop new tables: DROP TABLE IF EXISTS [5 new tables];
  3. Checkout previous: git checkout v3.4.0
  4. Rebuild: docker-compose -f docker-compose.staging.yml build api
  5. Restart: docker-compose -f docker-compose.staging.yml up -d api

Full Rollback with Database Restore:
  1. See docs/STAGING_DEPLOYMENT_PLAN.md (§ Rollback Procedures)
