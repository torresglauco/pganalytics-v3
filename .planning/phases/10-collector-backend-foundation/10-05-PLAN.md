From collector/src/replication_plugin.cpp (existing version detection):
```cpp
int PgReplicationCollector::detectPostgresVersion() {
    PGconn* conn = connectToDatabase("postgres");
    PGresult* res = PQexec(conn, "SELECT current_setting('server_version_num')::int");
    int version = std::stoi(PQgetvalue(res, 0, 0));
    postgres_version_major_ = version / 10000;
    postgres_version_minor_ = (version % 10000) / 100;
    return postgres_version_major_;
}

// Version-adaptive query
if (postgres_version_major_ >= 13) {
    query = "SELECT ... write_lag, flush_lag, replay_lag FROM pg_stat_replication";
} else {
    query = "SELECT ... 0 as write_lag_ms, 0 as flush_lag_ms, 0 as replay_lag_ms FROM pg_stat_replication";
}
```

PostgreSQL version capabilities:
- PG 11: Base streaming replication, pg_stat_replication
- PG 12: recovery.conf deprecated, standby.signal
- PG 13: write_lag, flush_lag, replay_lag in pg_stat_replication
- PG 14: pg_stat_wal_receiver improvements
- PG 15: pg_stat_subscription improvements
- PG 16: pg_stat_wal new view
- PG 17: Latest features

Collector deployment modes:
- Decentralized: Collector runs on same host as PostgreSQL, connects via Unix socket or localhost
- Centralized: Collector runs on remote host, connects via network to RDS/cloud PostgreSQL
- Mixed: Some collectors decentralized, some centralized

Collector authentication:
- mTLS: Certificate-based mutual authentication
- Token: Bearer token authentication
- API Key: Simple key-based auth
