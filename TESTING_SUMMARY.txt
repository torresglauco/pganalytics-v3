SILENT FAILURE PATTERN
================================================================================

Problem: E2E tests catch errors but don't fail
File: frontend/e2e/tests/05-user-management.spec.ts

Example (LINE 60-64):

    await expect(error.or(form)).toBeVisible({ timeout: 3000 });
  } catch {
    console.log('Email validation verified');  // ❌ SILENT FAIL
  }

Result: Test passes even if validation fails
Impact: No real test coverage, false confidence
Solution: Remove try/catch, let tests fail naturally

This pattern appears in multiple tests and is why user list breakage
wasn't caught before production.
