  <read_first>

  </read_first>

  <what_built>
    The unified CI workflow (05-01) is ready with a `ci-passed` status check that aggregates all quality gates. Now we need to configure branch protection to require this check.
  </what_built>

  <action>
    Configure branch protection rule for the `main` branch via GitHub UI or API:

    **Option A: GitHub UI (Recommended for visibility)**

    1. Navigate to: `https://github.com/{owner}/{repo}/settings/branches`
    2. Click "Add rule" or edit existing rule for `main`
    3. Configure:
       - Branch name pattern: `main`
       - Check "Require status checks to pass before merging"
       - Search for and select `ci-passed` from the status checks list
       - Check "Require branches to be up to date before merging"
       - Check "Include administrators" (ensures all users follow the rule)
       - Optionally: "Do not allow bypassing the above settings"
    4. Click "Create" or "Save changes"

    **Option B: GitHub CLI (if preferred)**

    ```bash
    gh api repos/{owner}/{repo}/branches/main/protection \
      --method PUT \
      --field required_status_checks='{"strict":true,"contexts":["ci-passed"]}' \
      --field enforce_admins=true \
      --field required_pull_request_reviews='{"dismiss_stale_reviews":true,"require_code_owner_reviews":false}' \
      --field restrictions=null
    ```

    **Option C: GitHub API directly**

    Use the example from RESEARCH.md (Pattern 3) with proper authentication.

    **Important considerations:**
    - The `ci-passed` status check must exist (from 05-01 workflow)
    - This rule applies to ALL PRs targeting main, including from admins
    - `strict: true` means branches must be up-to-date before merging

    **Do NOT:**
    - Do NOT add individual test jobs as required checks (use ci-passed only)
    - Do NOT skip "Include administrators" unless there's a specific reason
    - Do NOT configure protection for develop if it doesn't exist
  </action>

  <verify>
    <automated>gh api repos/{owner}/{repo}/branches/main/protection --jq '.required_status_checks.contexts' 2>/dev/null | grep -q "ci-passed" || echo "Branch protection verification requires GitHub CLI authentication"</automated>
  </verify>

  <done>
    - Branch protection rule created for main branch
    - `ci-passed` required status check configured
    - "Include administrators" enabled
    - Rule visible in GitHub repository settings
  </done>
